Professional Reflection

What was the problem you were solving in the projects for this course?

This project involved developing a C++ program for managing course data. The application reads course information from a file, organizes it into an appropriate data structure, and provides users with options to view courses in alphanumeric order, display details (including prerequisites), and efficiently search for specific courses. The primary challenge was creating an efficient and user-friendly system to store, retrieve, and display course information while ensuring proper data organization and usability. The project required sorting, searching, and structured data storage to facilitate quick and accurate retrieval of course details.

How did you approach the problem? Consider why data structures are important to understand.

I approached the problem by carefully analyzing the requirements and selecting an appropriate data structure. Understanding data structures is crucial because they directly impact the efficiency and performance of a program. For this project, I chose a vector to store course objects, which provided efficient sequential access and the ability to sort data. The vector structure allowed me to implement QuickSort for organizing courses alphabetically and facilitated efficient searching through the course list. This choice balanced memory efficiency with operational performance, demonstrating how proper data structure selection is fundamental to effective program design.

How did you overcome any roadblocks you encountered while going through the activities or project?

To overcome these roadblocks, I used debugging techniques such as print statements to verify data integrity and implemented a vector data structure and Course object structure to simplify data handling. Additionally, I reviewed best practices for sorting algorithms and ensured that prerequisites were mapped correctly. When transitioning to the enhanced version, I faced challenges in properly separating concerns and organizing code across multiple files. I overcame this by studying object-oriented design principles and refactoring the code incrementally, testing each change to ensure functionality was preserved.

How has your work on this project expanded your approach to designing software and developing programs?

This project has improved the way I write code by emphasizing clarity, modularity, and documentation. By applying these principles, I have developed a scalable and adaptable program that can be enhanced with additional features in the future. The enhancement phase particularly taught me the value of separating concerns and using classes to encapsulate related functionality. I now approach software design with a focus on maintainability and extensibility, considering how my code will be modified and extended by others (or myself) in the future. This experience has strengthened my understanding of both algorithmic efficiency and software architecture.

How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable?

Working on this project has fundamentally changed how I approach code organization and structure. The progression from ProjectTwo.cpp (a monolithic implementation) to CS_300_Enhancement (with separated CourseManager class) demonstrates the importance of maintainability. I now prioritize writing code that is easy to understand, modify, and extend. Using header files to separate interface from implementation, applying const-correctness, and using modern C++ features like lambdas for sorting have all become part of my standard practice. I've learned that readable code isn't just about comments—it's about clear structure, meaningful naming, and logical organization. The modular design makes the codebase more adaptable, allowing individual components to be modified without affecting the entire system.
