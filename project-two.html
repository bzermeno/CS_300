Project Two: Course Planner System

Original Implementation with Vector and QuickSort

Overview

Project Two is a C++ course management system demonstrating data structures and algorithm design principles. This is the original implementation showcasing fundamental programming concepts including file I/O, data structures, sorting algorithms, and interactive menu-driven interfaces.

Technology Stack: C++, Standard Template Library (STL)

Key Concepts: Data Structures (Vector), Sorting Algorithms (QuickSort), File I/O, String Manipulation

Date: February 23, 2025

GitHub: View Source Code

System Features

The course planner provides a menu-driven interface with the following capabilities:

1. Load Data Structure

Reads course information from a CSV file and stores it in memory:

  • Course Number: Unique identifier for each course
  • Course Name: Full title of the course
  • Prerequisites: List of required courses (can be none)

The system validates file access and provides error messages if the file cannot be opened.

2. Print Course List

Displays all courses in alphanumeric order:

  • Automatically sorts courses before display
  • Shows course number and course name
  • Presents a clean, organized schedule view

3. Print Course

Searches for and displays detailed information about a specific course:

  • Case-insensitive search functionality
  • Displays course number, name, and prerequisites
  • Handles courses with no prerequisites
  • Provides error message if course is not found

4. Exit

Gracefully terminates the program with a thank you message.

Data Structure Design

Course Structure

The system uses a custom struct to represent course data:

struct Course {
    string courseNumber;
    string courseName;
    vector<string> prerequisites;
};

This structure efficiently encapsulates all course-related information in a single, organized unit.

Vector Storage

Courses are stored in a vector container for several advantages:

  • Sequential Access: Efficient iteration through all courses
  • Dynamic Sizing: Automatically grows to accommodate any number of courses
  • Cache Friendly: Contiguous memory allocation improves performance
  • Random Access: Direct index-based access when needed

Algorithm Implementation

QuickSort Algorithm

The program implements QuickSort for efficient course sorting:

void quickSort(vector<Course>& courses, int left, int right) {
    int i = left, j = right;
    Course tmp;
    string pivot = courses[(left + right) / 2].courseNumber;
    
    // Partition
    while (i <= j) {
        while (courses[i].courseNumber < pivot)
            i++;
        while (courses[j].courseNumber > pivot)
            j--;
        if (i <= j) {
            tmp = courses[i];
            courses[i] = courses[j];
            courses[j] = tmp;
            i++;
            j--;
        }
    }

    // Recursion
    if (left < j)
        quickSort(courses, left, j);
    if (i < right)
        quickSort(courses, i, right);
}

Algorithm Analysis

  • Time Complexity: O(n log n) average case, O(n²) worst case
  • Space Complexity: O(log n) for recursion stack
  • In-place Sorting: Minimal additional memory usage
  • Unstable Sort: Equal elements may be reordered

QuickSort was chosen for its excellent average-case performance and efficiency with the vector data structure.

Search Algorithm

Linear search is used for course lookup:

  • Time Complexity: O(n) in worst case
  • Simple Implementation: Easy to understand and maintain
  • Case Insensitive: Converts search term to uppercase
  • First Match: Returns immediately upon finding course

File Parsing Implementation

The program reads and parses CSV data with robust error handling:

CSV Format

Expected file format: CourseNumber,CourseName,Prerequisite1,Prerequisite2,...

Example:

Parsing Logic

void readFile(string filename, vector<Course>& courses) {
    ifstream file(filename);
    if (!file) {
        cout << "Error opening file." << endl << endl;
        return;
    }
    string line;
    while (getline(file, line)) {
        Course course;
        string prereq;
        int pos = 0;
        pos = line.find(",");
        course.courseNumber = line.substr(0, pos);
        line.erase(0, pos + 1);
        pos = line.find(",");
        course.courseName = line.substr(0, pos);
        line.erase(0, pos + 1);
        while ((pos = line.find(",")) != string::npos) {
            prereq = line.substr(0, pos);
            course.prerequisites.push_back(prereq);
            line.erase(0, pos + 1);
        }
        course.prerequisites.push_back(line);
        courses.push_back(course);
    }
    cout << "Data loaded successfully." << endl << endl;
    file.close();
}

Technical Implementation Details

  • substr(): Extracts field values from strings
  • erase(): Removes processed data from string
  • transform(): Converts user input to uppercase for search

Error Handling

  • File access validation before reading
  • Clear error messages for user guidance
  • Invalid menu option handling
  • Course not found messaging

Code Organization

  • Separate functions for each major operation
  • Clear function documentation with comments

  • Main function handles program flow and user interaction

Performance Characteristics

Time Complexity Analysis

  • Load Data: O(n) - Linear scan of file
  • Sort Courses: O(n log n) - QuickSort average case
  • Search Course: O(n) - Linear search
  • Print Courses: O(n) - Iterate through all courses

Space Complexity

  • Vector Storage: O(n) - Stores all course data
  • QuickSort: O(log n) - Recursion stack depth
  • Auxiliary Space: O(1) - Fixed additional variables

Strengths and Limitations

Strengths

  • Simple, straightforward implementation
  • Efficient sorting with QuickSort
  • Clear, understandable code structure
  • Handles courses with multiple prerequisites
  • User-friendly interface

Limitations

  • All code in a single file (limited modularity)
  • No data persistence between sessions
  • Linear search could be optimized with binary search after sorting
  • Limited error handling for malformed CSV data
  • Global namespace usage with "using namespace std"

These limitations were addressed in the enhanced version. See the Enhancement page for details.
