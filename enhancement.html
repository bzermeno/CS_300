CourseManager Class

The new CourseManager class encapsulates all course management functionality:

Class Definition (CourseManager.h)

#ifndef COURSE_MANAGER_H
#define COURSE_MANAGER_H

#include <string>
#include <vector>

struct Course {
    std::string courseNumber;
    std::string courseName;
    std::vector<std::string> prerequisites;
};

class CourseManager {
private:
    std::vector<Course> courses;

public:
    void readFile(const std::string& filename);
    void printCourses() const;
    void courseSearch(const std::string& courseNumber) const;
};

#endif // COURSE_MANAGER_H

Key Improvements

  • Encapsulation: Course data is private to the class
  • Clear Interface: Public methods define how to interact with courses
  • Const-Correctness: Read-only methods marked as const
  • Include Guards: Prevents multiple inclusion problems
  • Explicit Namespace: Uses std:: instead of "using namespace std"

Enhanced Implementation

Improved File Reading

The readFile method now includes automatic sorting and better validation:

void CourseManager::readFile(const std::string& filename) {
    std::ifstream file(filename);
    if (!file) {
        std::cout << "Error opening file.\\n\\n";
        return;
    }

    courses.clear();  // Clear existing data

    std::string line;
    while (getline(file, line)) {
        Course course;

        size_t pos = line.find(',');
        course.courseNumber = line.substr(0, pos);
        line.erase(0, pos + 1);

        pos = line.find(',');
        course.courseName = line.substr(0, pos);
        line.erase(0, pos + 1);

        while ((pos = line.find(',')) != std::string::npos) {
            course.prerequisites.push_back(line.substr(0, pos));
            line.erase(0, pos + 1);
        }

        if (!line.empty()) {
            course.prerequisites.push_back(line);
        }

        courses.push_back(course);
    }

    std::cout << "Data loaded successfully.\\n\\n";

    // Sort courses once loaded using STL algorithm
    std::sort(courses.begin(), courses.end(),
        [](const Course& a, const Course& b) {
            return a.courseNumber < b.courseNumber;
        });
}

Key Changes

  • Clear Before Load: Safely handles multiple file loads
  • Empty Check: Prevents adding empty prerequisites
  • Automatic Sorting: Sorts once during load instead of each print
  • STL Algorithm: Uses std::sort with lambda comparator
  • size_t: Proper type for string positions

Modern C++ Practices

1. Lambda Functions

Uses lambda expressions for inline comparison logic:

std::sort(courses.begin(), courses.end(),
    [](const Course& a, const Course& b) {
        return a.courseNumber < b.courseNumber;
    });

This eliminates the need for a separate 100-line QuickSort implementation.

2. Const-Correctness

Methods that don't modify state are marked const:

void printCourses() const;
void courseSearch(const std::string& courseNumber) const;

This enables compiler optimizations and prevents accidental modifications.

3. Range-Based For Loops

Cleaner iteration over collections:

for (const auto& course : courses) {
    std::cout << course.courseNumber << ", "
              << course.courseName << std::endl;
}

4. Improved Error Handling

Better validation and user-friendly error messages:

if (courses.empty()) {
    std::cout << "Course data not loaded.\\n\\n";
    return;
}

Enhanced Course Search

The search implementation is cleaner and more robust:

void CourseManager::courseSearch(const std::string& courseNumber) const {
    if (courses.empty()) {
        std::cout << "Course data not loaded.\\n\\n";
        return;
    }

    for (const auto& course : courses) {
        if (course.courseNumber == courseNumber) {
            std::cout << course.courseNumber << ", "
                      << course.courseName << '\\n';

            if (course.prerequisites.empty()) {
                std::cout << "Prerequisites: None\\n\\n";
            }
            else {
                std::cout << "Prerequisites: ";
                for (size_t i = 0; i < course.prerequisites.size(); ++i) {
                    if (i > 0) std::cout << ", ";
                    std::cout << course.prerequisites[i];
                }
                std::cout << "\\n\\n";
            }
            return;
        }
    }

    std::cout << "Course not found.\\n\\n";
}

Improvements

  • Checks if data is loaded before searching
  • Uses empty() instead of checking string length
  • Cleaner prerequisite formatting logic
  • More readable with explicit loop indexing

Separation of Concerns

The enhanced architecture separates responsibilities:

Main Program (CS_300_Enhancement.cpp)

  • Responsibility: User interface and program flow
  • Does: Display menus, get input, call CourseManager methods
  • Doesn't Do: File parsing, data storage, course management

CourseManager Class

  • Responsibility: Course data management
  • Does: Load files, store courses, search and display courses
  • Doesn't Do: User interaction, menu display

Benefits

  • Easy to modify user interface without touching course logic
  • CourseManager can be reused in other programs
  • Testing individual components is simpler
  • Multiple developers can work on different parts

Lessons Learned

Object-Oriented Design

Encapsulating related data and functionality in classes makes code more organized and reusable. The CourseManager class provides a clear interface while hiding implementation details.

Separation of Concerns

Separating user interface from business logic makes both easier to modify and test. The main program doesn't need to know how courses are stored or sorted.

Modern C++ Features

Using STL algorithms and lambda functions results in more concise, readable, and maintainable code. The standard library provides well-tested, optimized implementations.

Header Files

Separating interface (header) from implementation (cpp file) is a fundamental C++ practice that enables separate compilation and clearer code organization.
